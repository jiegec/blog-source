Caveats & Disclaimer (click to expand)
  1. Scores can only be used to compare performance of CPUs with the same ISA when variables are controlled (i.e., measured with the same binary as commonly said; additional variables that affect performance are listed below). This allows comparison through test results for:
    1. Performance of Intel and AMD CPUs on the AMD64 ISA
    2. Performance of Apple, ARM, Huawei, and Qualcomm CPUs on the ARM64 ISA
    3. Performance of different CPUs on the LoongArch ISA
  2. Using scores to compare CPUs across different ISAs is less convincing.
  3. Even with identical hardware, the following factors can significantly affect results:
    1. Different compilers (e.g., GCC is usually faster than Clang with the same compilation flags)
    2. Different compiler versions (newer versions are usually faster, but there are exceptions)
    3. Different compiler flags (e.g., whether LTO is enabled, whether -march=native is set)
    4. Different memory allocator implementations (e.g., libc's built-in malloc or jemalloc)
    5. Different C standard library implementations (e.g., glibc or musl)
    6. Different frequency scaling, scheduler, or CPU affinity policies (e.g., improper pinning can drop frequency from 4.0 GHz to 3.4 GHz)
    7. Different kernel versions (some versions can significantly degrade performance)

In short, scores given without complete test environment information are unreliable.

If you need to cite any test results on this page, please always list the test environment used for that result and make sure you have a thorough understanding of the above caveats.
